003    /**
004     * {@link Operation} is base Interface which must
005     * be implemented by all classes representing concrete
006     * database operations (i.e. SQL statements).
007     * <p>
008     * The SQL statement represented by an {@link Operation}
009     * can either be supplied by annotating the concrete
010     * Operation implementation with
011     * {@link org.phlo.tuplez.operation.Statement}, or by
012     * implementing the interface {@link StatementIsComputed}
013     * in addition to {@link Operation}.
014     * <p>
015     * For each (named) parameter in the SQL statement, the InputType
016     * must define a corresponding getter method (following
017     * the usual Java Bean naming convention, i.e. the getter
018     * for the parameter :myValue has to be called getMyValue()).
019     * <p>
020     * Similarly, for each column of the statement's result,
021     * the OutputType must contain a getter. The name to column
022     * matching is more lenient than in the input case - the
023     * getter's name must get{@literal <columnName>} after both
024     * strings have been down-cased and all non-alphanumeric
025     * characters have been removed.
026     * <p>
027     * The OutputType's getter's return type determined the
028     * JDBC ResultSet accessor method used to extract the output
029     * value.
030     * <p>
031     * The mapping of column names to OutputType getters and
032     * to ResultSet accessors can be overriden by annotating
033     * concrete database operation classes with
034     * {@link org.phlo.tuplez.operation.OutputMapping}.
035     * <p>
036     * {@link Operation} extends {@link OperationInput} and
037     * {@link OperationOutput}, each parameterized with the
038     * {@literal InputType} respectively the {@literal OutputType}.
039     * This allows other parameterized interfaces like
040     * {@link GeneratesKey} to force a particular
041     * {@literal InputType} or {@literal OutputType}.
042     * 
043     * @see OperationInput
044     * @see OperationOutput
045     * @see org.phlo.tuplez.operation.Statement
046     * @see org.phlo.tuplez.operation.OutputMapping
047     *
048     * @param <InputType> Interface representing a database operation's input parameters
049     * @param <OutputType> Interface representing a database operation's output parameters
050     */
051    public interface Operation<InputType, OutputType> extends OperationInput<InputType>, OperationOutput<OutputType> {
052            /* This interface is just a means of achieving
053             * compile-time type safety
054             */ 
055    }
